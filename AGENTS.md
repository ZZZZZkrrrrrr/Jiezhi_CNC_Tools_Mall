# 项目级代理指令：数控精密刀具商城网页端复刻

## 项目目标

将墨刀分享项目 `https://modao.cc/ai/share/project/6a5ee1fe75d0275ef6b40be0` 完整迁移为可运行的桌面网页端。参考项目是唯一的页面、文案、视觉和交互事实来源。

## 不可违反的规则

1. 开发前必须完成参考项目采集，不得先凭经验搭商城模板。
2. 参考项目中出现的页面、弹窗、状态、入口、文案、图片、图标和跳转关系不得遗漏。
3. 禁止用截图、iframe 或大面积绝对定位冒充界面实现。
4. 禁止只完成首页；所有可达页面都必须实现。
5. 禁止把小程序窄屏界面直接居中放到桌面网页中。
6. 先建立 390px 参考校验模式，再实现 1440px 桌面适配模式。
7. 图片优先从参考项目提取。缺失图片由代理联网搜索、下载、登记来源并本地化，不得长期热链。
8. 任何从互联网获得的图片、Logo、参数和文案必须写入 `docs/asset-manifest.csv` 或 `docs/source-manifest.md`。
9. 不确定项不得静默猜测。写入 `docs/assumptions.md`，采用最保守、最接近参考项目的方案继续执行。
10. 除非参考链接完全无法访问、需要密码或出现人机验证，否则不得中途要求用户逐页确认。
11. 每完成一个阶段必须执行对应验收命令，并将结果写入 `docs/progress.md`。
12. 最终提交前必须完成 Playwright E2E、响应式检查、无障碍检查和截图差异检查。

## 实现技术

- React + TypeScript + Vite
- React Router
- SCSS Modules + CSS Variables
- Zustand（仅在跨页面状态确有必要时）
- React Hook Form + Zod
- i18next
- Playwright
- Vitest
- Sharp
- ESLint + Prettier

不得引入会显著改变视觉语言的大型 UI 组件库。通用组件自行实现。

## 目录约定

- `reference/`：参考项目截图、网络资源、流程和清单
- `src/pages/`：路由级页面
- `src/components/`：通用组件
- `src/features/`：搜索、购物车、询价、账户等业务模块
- `src/data/`：Mock 数据及数据映射
- `src/assets/`：本地化素材
- `src/styles/`：设计变量和全局样式
- `tests/e2e/`：端到端测试
- `docs/`：采集、差异、来源、决策和验收文档
- `scripts/`：采集、图片处理、视觉回归脚本

## 开发质量要求

- 所有页面必须有唯一稳定路由。
- 所有按钮、卡片、Tab、筛选器和弹窗必须有可测试选择器 `data-testid`。
- 所有图片必须有宽高、替代文本和加载失败兜底。
- 所有列表必须覆盖加载、空数据、错误和正常状态。
- 所有表单必须覆盖必填、格式错误、提交中、成功和失败状态。
- 文案必须进入语言资源文件，不得散落硬编码。
- 商品参数不得凭空冒充真实性能。无法确认的字段使用明确的演示数据标识。
- 每次提交保持单一目的，禁止在一个提交中混杂采集、重构和视觉修复。

## 阶段门禁

未满足以下条件不得进入下一阶段：

- 参考采集完成：`reference/reference-manifest.json` 存在，页面矩阵和交互矩阵非空。
- 素材完成：所有页面需要的素材均有本地文件或明确占位替换计划。
- 页面完成：页面矩阵中的 `implementation_status` 全部为 `done`。
- 交互完成：交互矩阵中的 `test_status` 全部为 `passed`。
- 视觉完成：关键页面不存在明显错位、溢出、拉伸和风格漂移。
- 最终完成：`npm run build`、`npm run test`、`npm run test:e2e` 全部通过。
