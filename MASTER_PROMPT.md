# Codex 主执行提示词

你现在负责将下方墨刀小程序原型完整复刻为桌面网页端：

`https://modao.cc/ai/share/project/6a5ee1fe75d0275ef6b40be0`

这是一个数控精密刀具 B2B 商城项目，当前主要服务中国大陆客户，并需要预留英文、美元、国际地址和外贸询价扩展能力。

你必须自主完成参考项目采集、页面分析、素材搜索、素材下载、网页开发、交互实现、测试和视觉校验。不要只给方案，不要只做首页，不要要求用户逐页提供截图。除非链接被密码、人机验证或权限完全阻断，否则持续执行到项目可运行并通过验收。

## 第一原则

参考项目有什么，网页端就必须有什么。页面、入口、文案、素材、弹窗、状态、交互和跳转关系均以参考项目为准。你只能改变适配桌面网页所必需的布局形式，不能改变项目本身的信息架构和视觉身份。

“一比一”采用双层定义：

1. **参考校验模式**：在参考项目原始小程序宽度附近（优先自动识别，无法识别时使用 390×844）实现像素级接近，用来证明页面内容和视觉细节没有漏项。
2. **桌面网页模式**：在 1440×900、1920×1080 下，把同一内容迁移为真正的桌面网页布局。不得放入手机外框，不得把 390px 页面简单拉伸或居中。

## 强制执行顺序

### 阶段 A：环境与访问检查

1. 阅读仓库根目录 `AGENTS.md`。
2. 检查当前 Codex 是否具备 live web search、工作区写入和命令网络访问。
3. 安装并验证 Chromium/Playwright。
4. 打开参考链接，等待动态资源加载。
5. 如果首次打开为空白，依次尝试：
   - 等待 10 秒并记录控制台；
   - 使用桌面与移动 User-Agent；
   - 禁用缓存后重新加载；
   - 记录所有 fetch/XHR/WebSocket；
   - 检查页面内嵌 JSON、localStorage、sessionStorage、IndexedDB；
   - 检查 iframe、canvas、shadow DOM；
   - 截取整页和视口截图。
6. 只有在确认出现登录、访问密码、人机验证或链接失效时，才生成 `docs/access-blocker.md`。其他抓取困难不得作为停止理由。

### 阶段 B：参考项目完整采集

1. 保存首屏截图、整页截图、DOM 快照、可访问性树、控制台日志、网络日志和可见文本。
2. 提取所有可点击元素：导航、Tab、卡片、按钮、图标按钮、轮播、筛选、输入框、底部导航和热区。
3. 采用广度优先方式遍历页面和状态。每次操作后对比 URL、标题、可见文本、DOM 指纹和截图哈希，发现新状态即保存。
4. 对无 DOM 的 Canvas 原型：
   - 保存高分辨率截图；
   - 根据视觉区域生成候选点击网格；
   - 逐个点击并检测截图变化；
   - 记录坐标、来源状态和目标状态；
   - 对返回、关闭、TabBar 等固定区域优先探索。
5. 对弹窗、抽屉、下拉、选择器、Toast、确认框、登录态和空状态分别采集。
6. 对涉及真实提交、支付或删除的操作，使用演示数据并停在确认前；不得执行真实交易。
7. 生成：
   - `reference/reference-manifest.json`
   - `docs/page-matrix.csv`
   - `docs/interaction-matrix.csv`
   - `docs/reference-audit.md`
   - `reference/screenshots/*`
   - `reference/flows/*`
8. 页面矩阵至少包含：页面 ID、原型状态 ID、页面名称、入口、页面类型、主要模块、可点击元素、目标状态、是否弹窗、是否需要登录、截图路径、实现路由、完成状态。
9. 交互矩阵至少包含：来源页面、控件、触发方式、前置状态、目标页面或状态、动画、错误状态、测试用例、测试状态。

### 阶段 C：视觉规范提取

从截图、DOM 和资源中提取：

- 品牌主色、辅助色、背景色、文本色、边框色、成功/警告/错误色；
- 字体、字号、字重、行高、字距；
- 按钮、输入框、卡片、弹窗、标签的尺寸、圆角和阴影；
- 页面间距、容器宽度、栅格、图片比例；
- 图标风格和线宽；
- 导航选中态、悬停态、禁用态；
- 商品卡片和商品详情的字段顺序。

将结果写入：

- `src/styles/tokens.css`
- `src/styles/global.scss`
- `docs/design-system.md`
- `docs/design-tokens.json`

禁止在不同页面随意使用新颜色或新尺寸。

### 阶段 D：图片与素材

素材优先级：

1. 参考项目网络请求中可直接取得的原图；
2. 参考项目 DOM/CSS 中的图片地址、SVG、iconfont 和背景图；
3. 参考项目截图中可无损裁切的装饰素材，只能作为临时补位；
4. 官方刀具制造商网站、官方产品目录、官方媒体资料；
5. 获授权的经销商或行业产品数据库；
6. 具备明确商用许可的图库；
7. 最后才使用自制的中性占位素材。

必须自行联网搜索缺失图片。推荐检索域：

- `zccct.com`
- `gesac.com.cn`
- `huareal.com.cn`
- `sandvik.coromant.com`
- `kennametal.com`
- `mitsubishicarbide.com`
- `secotools.com`

推荐搜索词组合：

- 数控车削刀片 白底 官方
- 硬质合金立铣刀 产品图 官方
- 可转位面铣刀盘 白底
- 内冷硬质合金钻头 产品图
- ER 高精度筒夹 白底
- HSK 液压刀柄 产品图
- CNC turning insert official product image
- solid carbide end mill official product image
- indexable face mill official image
- carbide drill official product image
- hydraulic chuck HSK official image

每一项下载素材必须登记到 `docs/asset-manifest.csv`，字段包括：

`asset_id, local_path, purpose, category, source_url, source_domain, source_title, original_filename, downloaded_at, copyright_status, replacement_required, notes`

规则：

- 不得热链；全部下载到 `src/assets/`。
- 不得把其他品牌 Logo 当作本项目品牌 Logo。
- 原型中若有明确品牌，保持原样；若没有，使用项目自身中性品牌，不擅自植入第三方品牌。
- 发布前无法确认授权的素材标记 `replacement_required=true`。
- 商品名称、外形和图片必须对应，不得用钻头图片表示铣刀，不得用刀柄表示刀片。
- 使用 Sharp 生成 WebP/AVIF 和响应式尺寸。
- 商品卡片图优先 1:1，详情主图优先 1:1 或参考项目比例，Banner 按参考项目比例。
- 透明 PNG 保留透明背景；白底图统一为纯白或参考项目背景色。

### 阶段 E：网页信息架构迁移

先保持所有内容和操作，再转换布局：

- 小程序 TabBar → 桌面顶部主导航或左侧导航，名称、顺序和选中态不变。
- 首页纵向模块 → 在不改变模块顺序的前提下转为横向 Banner、多列入口和商品网格。
- 分类页 → 左侧分类树/筛选，右侧结果区；小屏恢复抽屉。
- 商品列表 → 顶部结果信息和排序、左侧筛选、右侧 3–5 列商品卡片。
- 商品详情 → 左图右信息，下方详情/参数/资料区域。
- 购物车或询价单 → 桌面表格或宽卡片，保留原字段与操作。
- 个人中心 → 左侧菜单、右侧内容。
- 小程序底部弹层 → 居中 Dialog、右侧 Drawer 或锚定 Popover，选项和结果不变。

桌面基准：

- 1440×900 主要验收；
- 1920×1080 次要验收；
- 内容最大宽度 1280–1360px，最终值从参考视觉密度推导；
- 页面左右安全边距不低于 32px；
- 采用 12 栅格或等效布局；
- 1024px 以上不得出现手机窄列；
- 768–1023px 为平板；
- 小于 768px 恢复移动网页布局，并作为参考校验模式。

- 地址管理

- 客服/联系
- 企业介绍
- 帮助或售后

不得把这份候选清单当作替代参考采集；以页面矩阵最终结果为准。

商品分类和演示数据必须专业。参考项目缺少内容时，可从以下结构中补足，但不得覆盖已有分类：

- 车削刀具：外圆、内孔、切断、切槽、螺纹、仿形、车削刀片；
- 铣削刀具：立铣、面铣、球头、圆鼻、T 型槽、燕尾槽、可转位铣削、铣削刀片；
- 孔加工：麻花钻、中心钻、阶梯钻、深孔钻、可转位钻、铰刀、镗刀、沉头刀；
- 螺纹加工：丝锥、挤压丝锥、螺纹铣刀、板牙、螺纹刀片；

- 刀片：车削、铣削、切槽、螺纹、钻削、CBN、PCD、陶瓷；

核心数据模型至少包括：商品 ID、中英文名、型号、分类、品牌、图片、适用材料、刀具材料、涂层、直径、刃数、柄径、刃长、总长、精度、库存、价格显示类型、MOQ、单位、交期、购买/询价能力和状态。

- 导航、返回、Tab、轮播；
- 搜索建议、历史和清除；
- 分类切换、多选筛选、重置、排序、分页；
- 商品图切换和放大；
- 规格选择、数量增减；
- 收藏、取消收藏；
- 加入购物车/询价单；
- 购物车选择、全选、编辑、删除、汇总；
- 表单校验和提交反馈；
- 登录态切换；

购物车、收藏、最近浏览和演示登录态使用 LocalStorage 持久化。数据访问封装为 service 层，便于替换真实 API。

### 阶段 I：工程实现要求

使用：React、TypeScript、Vite、React Router、SCSS Modules、CSS Variables、i18next、Playwright、Vitest。

禁止：

- 单文件堆砌；
- 截图替代组件；
- iframe 嵌入墨刀；
- Canvas 绘制整站；
- 全局绝对定位；
- 大型 UI 库默认皮肤；
- 随机占位图片；
- 与参考项目无关的促销模块；
- 未登记来源的互联网素材；
- 无响应的假按钮。

每个交互元素加 `data-testid`。每个路由设置页面标题。图片懒加载、错误兜底和合理尺寸。避免布局偏移。

### 阶段 J：测试与视觉验收

建立三层测试：

1. **功能测试**：Vitest 验证状态、工具函数和数据映射。
2. **流程测试**：Playwright 覆盖导航、搜索、筛选、详情、收藏、购物车/询价、登录和个人中心。
3. **视觉测试**：关键页面在参考宽度、1440px、1920px 截图，与参考截图或批准基线比较。

视觉验收：

- 参考宽度模式逐页对比；
- 桌面模式模块顺序、视觉语言、字段和操作不变；
- 不允许文字截断、图片拉伸、按钮错位、遮挡和异常横向滚动；
- 页面间颜色、字号、圆角、阴影一致；
- 关键页面差异图保存到 `reference/diffs/`；
- 自动差异只能辅助判断，必须再做人工截图检查。

每个页面至少验证：正常、加载、空数据和错误状态。每个表单至少验证成功和校验失败。

运行：

```bash
npm run lint
npm run test
npm run test:e2e
npm run build
```

## 不得提前结束的条件

以下任一情况存在时不得宣布完成：

- 页面矩阵仍有未实现项；
- 存在点击无反应；
- 存在未登记来源素材；
- 关键页面仍使用随意占位图；
- 仅完成首页或主要页面；
- 1440px 仍是窄屏小程序布局；
- 构建或测试失败；
- 未生成最终验收报告。
